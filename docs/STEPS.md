## Step 3 — Ingestion: Walker & Parser & Chunker

**Goal:** Given a repository path, produce a list of `IndexChunk` objects ready for embedding.

### Files to create
- `src/repolens/ingestion/walker.py`
  - `GitWalker(repo_path, ignore_filter)` — yields `(path, language, content)` for all eligible files
  - `get_changed_files(repo_path, since_sha)` — for incremental re-indexing
  - Language detection by file extension → returns `None` for unsupported files (skipped silently)
- `src/repolens/ingestion/parser.py`
  - `TreeSitterParser` — loads grammar for each language, extracts semantic nodes
  - Extracts: functions, classes, structs, interfaces, methods
  - Returns `List[ParsedChunk]` with: file_path, symbol_name, symbol_type, signature, docstring, body, start_line, end_line, language
  - Falls back to whole-file chunking for unsupported node types
- `src/repolens/ingestion/chunker.py`
  - `SemanticChunker(max_tokens, overlap_tokens)` — receives `ParsedChunk`, returns `List[IndexChunk]`
  - Sub-chunks large functions with sliding window (never splits inside a nested block)
  - Drops chunks below minimum token count
  - `IndexChunk` dataclass: all fields from `ParsedChunk` + `chunk_id` (uuid4) + `token_count`

### Tests
- `tests/unit/test_parser.py` — test parsing a Python file, a Go file; verify symbol extraction
- `tests/unit/test_chunker.py` — test that a 1000-token function is correctly sub-chunked
- Add a `fixtures/` directory under `tests/` with small sample files for each language

### Commit
```
feat(ingestion): add git walker, tree-sitter parser, and semantic chunker
```

### Definition of done
- Parse a real Python file → get back a list of `ParsedChunk` objects with correct line numbers
- Parse a real Go file → same
- A 1000-token function is sub-chunked correctly with overlap
- All tests pass

## Step 4 — Ingestion: Symbol Graph, Embedder & BM25

**Goal:** Complete the ingestion pipeline. Given `List[IndexChunk]`, produce embeddings, a BM25
index, and a symbol graph.

### Files to create
- `src/repolens/ingestion/graph.py`
  - `SymbolGraphBuilder` — walks chunks and extracts call edges and import edges
  - Uses tree-sitter to find function call nodes within each chunk's body
  - Returns `nx.DiGraph` with nodes = `chunk_id`, edges = `{type: "calls"|"imports"|"implements"}`
- `src/repolens/ingestion/embedder.py`
  - `CodeEmbedder(model_name, cache)` — loads jina-v2-base-code via sentence-transformers
  - `embed(chunks: List[IndexChunk]) -> np.ndarray` — embeds in batches of 32
  - Checks disk cache first; only embeds chunks whose content hash is not cached
- `src/repolens/ingestion/bm25.py`
  - `BM25Indexer` — tokenises chunks, builds BM25Okapi index, serialises to disk
  - `search(query: str, top_k: int) -> List[Tuple[chunk_id, score]]`
  - Tokeniser: whitespace split + CamelCase decomposition (so "handleRoute" → ["handle", "Route"])

### Commit
```
feat(ingestion): add symbol graph builder, code embedder, and BM25 indexer
```

### Definition of done
- `CodeEmbedder.embed([chunk])` returns a numpy array of shape `(1, 768)`
- Same chunk embedded twice uses cache on second call (verified via cache hit counter)
- `BM25Indexer.search("handle route")` returns relevant chunks
- Symbol graph has edges between a function and the functions it calls

## Step 6 — Retrieval Engine

**Goal:** Given a query string, return the most relevant code chunks from the index.

### Files to create
- `src/repolens/retrieval/hybrid.py`
  - `HybridRetriever(vector_store, bm25_indexer, embedder, llm_client)`
  - `retrieve(query, top_k=20) -> List[IndexChunk]`
  - Runs HyDE: generates a hypothetical code snippet, embeds it
  - Runs dense ANN search + BM25 search in parallel (asyncio.gather)
  - Fuses with RRF (`k=60`): `score = Σ 1/(60 + rank_i)`
  - Returns top-K=20 by RRF score
- `src/repolens/retrieval/reranker.py`
  - `CrossEncoderReranker(model_name)` — loads ms-marco-MiniLM-L-6-v2
  - `rerank(query, chunks: List[IndexChunk], top_k=8) -> List[IndexChunk]`
  - Scores each (query, chunk.body_preview) pair and returns top-K by score
- `src/repolens/retrieval/expander.py`
  - `GraphExpander(graph_store, vector_store)`
  - `expand(chunks: List[IndexChunk], hops=1) -> List[IndexChunk]`
  - For each chunk, fetches 1-hop neighbours from the graph store
  - Fetches those chunks from the vector store
  - Deduplicates, caps total at 12 chunks

### Tests
- `tests/unit/test_hybrid.py` — test RRF fusion with mock dense and BM25 results
- `tests/integration/test_retrieval.py` — run full retrieval pipeline on the test repo from Step 5

### Commit

### Definition of done
- `HybridRetriever.retrieve("how does authentication work")` returns relevant chunks

## Step 7 — Generation + Verification

**Goal:** Given retrieved chunks, produce a grounded, verified answer with a grounding score.

### Files to create

  - `BaseLLMClient` (ABC) with `async def stream(messages, system) -> AsyncIterator[str]`
- `src/repolens/generation/llm/anthropic.py` — `AnthropicClient(api_key, model)`
- `src/repolens/generation/llm/openai.py` — `OpenAIClient(api_key, model)`
- `src/repolens/generation/llm/ollama.py` — `OllamaClient(base_url, model)`
- `src/repolens/generation/prompt.py`
  - `build_system_prompt() -> str` — the grounded system prompt (forces citations, null path)
  - `build_user_message(query, chunks) -> str` — formats chunks with file path, lines, body
- `src/repolens/generation/validator.py`
  - `CitationValidator(repo_path)`
  - `validate(answer_text, citations) -> ValidationResult`
  - For each citation: open file, check line range, compute similarity to answer sentence
  - Returns: `ok: bool`, `invalid_citations: list`, `reason: str`
- `src/repolens/generation/scorer.py`
  - `GroundingScorer(model_name)` — loads nli-deberta-v3-small
  - `score(answer_text, cited_chunks) -> GroundingResult`
  - Splits answer into sentences; NLI each against its cited chunk
  - Returns: `score: float`, `verdict: "high"|"medium"|"low"|"none"`

### Tests
- `tests/unit/test_scorer.py` — test that a clearly grounded sentence scores "high"
- `tests/unit/test_validator.py` — test that a bad line range is caught

  - `create_app() -> FastAPI` — factory function

- `src/repolens/api/repos.py` — `POST /api/repos`, `GET /api/repos`, `GET /api/repos/{id}`,
